# Internationalization

Read `.algolab/config.json` `i18n` before creating user-facing content.

Fields:

- `i18n.ui_locale`: default language for replies and optional review notes
- `i18n.problem_locale`: default language for new problem statements
- `i18n.fallback_locale`: fallback language
- `meta.json.locale`: language used by a specific problem

Localize:

- problem statements
- hints
- explanations
- reviews
- test analysis
- final summaries

Keep machine fields stable in English:

- JSON keys
- file names
- directory names
- slugs
- statuses
- compare modes
- Python function names

If the user requests a language for a single problem or session, follow that request and record it in `meta.json.locale` or the relevant record.
